# 发放明细页面 (`issuance`)

## 1. 页面目的

此页面是订单通过平台风控后的最终发放确认环节。此阶段所有财务数据均为**只读**，不可修改。页面的核心目的是让用户进行最终核对，检查账户余额，并在确认一切无误后，授权执行批量付款。

## 2. 功能模块详解

### 2.1 流程步骤器 (Stepper)

-   **作用**: 标记订单已进入发放流程，并显示当前所处阶段。
-   **步骤**: `业务确认` (已完成) -> `平台风控` (已完成) -> `确认发放` (当前) -> `发放中` (待办) -> `发放完成` (待办)。
-   **UI**: `平台风控` 步骤明确标记为已完成，`确认发放` 为当前步骤。

### 2.2 发放摘要 (Issuance Summary)

-   **发放信息**: 清晰地展示本次发放的内容批次、月份和任务，以及对应的款项类型 (例如, "经营所得")。
-   **信息提示框**:
    -   明确告知用户此订单已通过平台风控，可以继续发放。
    -   警示用户在发放前需确认账户余额，并说明点击 "确认发放" 后资金将被冻结。

### 2.3 财务摘要卡片 (Financial Summary)

-   **目的**: 与业务确认页保持一致，方便用户快速核对订单的核心财务数据。
-   **核心指标卡片**:
    -   **订单总金额**
    -   **应发总金额**
    -   **总服务费**
    -   **总税费**

### 2.4 接收人数据表 (Recipient Data Table)

-   **功能**: 以**只读**模式列出所有款项接收人及其最终的支付金额。
-   **列定义**:
    -   `姓名`: 接收人姓名及其状态 (例如, "正常")。
    -   `人员信息`: 显示银行、账号、手机号和加***的身份证号。悬停可查看完整信息。
    -   `应发总金额`: 最终确认的税前总金额。
    -   `税后总金额`: 最终确认的实发到手金额。
    -   `订单总金额`: 最终确认的单人订单总成本。
    -   `操作`:
        -   `详细` 按钮: 点击后打开 "只读详细信息弹窗"。

### 2.5 只读详细信息弹窗 (Read-Only Details Modal)

-   **触发**: 点击接收人数据表中对应行的 "详细" 按钮。
-   **内容**: 显示与上一页弹窗完全相同的费用明细分类，但所有字段均为**只读**文本，不可编辑。
-   **操作**:
    -   `关闭` 按钮: 关闭弹窗。

### 2.6 页面操作栏 (Page Actions)

-   **左侧 (次要操作)**:
    -   `下载付款通知书`: 下载给接收人的正式付款通知文件。
    -   `下载支付明细`: 下载最终确认的支付明细报表。
    -   `作废`: 取消本次发放，此为危险操作。
-   **右侧 (主要操作与状态)**:
    -   **账户余额信息**:
        -   `账户可用余额`: 实时显示当前账户的可用资金。
        -   `还需充值`: 根据订单总金额和可用余额计算出的资金缺口，若余额不足则高亮显示（红色）。
    -   **操作按钮**:
        -   `充值`: 跳转到账户充值页面。
        -   `返回列表`: 放弃操作，返回订单列表页。
        -   `确认发放`: 核心操作按钮。点击后，系统将执行后续的付款流程。
