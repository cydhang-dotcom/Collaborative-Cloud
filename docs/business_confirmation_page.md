# 业务确认页面 (`confirmation`)

## 1. 页面目的

这是在订单进入支付阶段之前，用于审核和确认新业务订单的核心页面。业务人员在此页面可以详细审查每位收款人的财务细节，按需编辑各项金额，并在确认无误后将订单推进到下一个流程。

## 2. 功能模块详解

### 2.1 流程步骤器 (Stepper)

-   **作用**: 直观地展示用户在整个订单确认工作流中的当前位置。
-   **步骤**: `创建订单` (已完成) -> `业务确认` (当前) -> `付款确认` (待办) -> `完成` (待办)。
-   **UI**: 已完成的步骤为绿色，当前步骤为高亮绿色，待办步骤为灰色。

### 2.2 订单摘要 (Order Summary)

-   **订单信息**: 清晰地展示订单所属的月份 (`2025-11`) 和具体的结算任务 (`云才网络-UI设计服务`)。
-   **核心指标卡片**:
    -   **订单总金额**: 整个订单的成本总计。
    -   **应发总金额**: 所有人员税前收入的总和。
    -   **总服务费**: 平台收取的服务费总计。
    -   **总税费**: 订单涉及的个人所得税和增值税的总和。

### 2.3 人员数据表 (Personnel Data Table)

-   **功能**: 以表格形式列出与订单相关的所有人员及其详细的财务构成。
-   **列定义**:
    -   `姓名`: 人员姓名。
    -   `人员信息`:
        -   默认显示：银行、完整的银行账号、手机号和加***的身份证号。
        -   鼠标悬停时，通过浮层工具提示 (Tooltip) 显示完整的身份证号、手机号、任务、开户行和收款账号。
    -   `个人所得明细` (可编辑):
        -   `劳务报酬部分`: 可点击编辑的数值。
        -   `经营所得部分`: 可点击编辑的数值。
        -   `个体户部分`: 可点击编辑的数值。
        -   `应发总金额`: 可点击编辑的数值。编辑此项后，其变化会按原始比例自动分配到前三项子金额中。
    -   `平台及税务费用` (自动计算):
        -   `服务费`: 根据应发总金额自动计算 (`应发总金额 * 5%`)。
        -   `个税`: 根据劳务报酬部分自动计算 (`劳务报酬部分 * 10%`)。
        -   `增值税`: 根据经营所得和个体户部分自动计算 (`(经营所得部分 + 个体户部分) * 3%`)。
    -   `税后总金额`: 自动计算的最终实发金额。
    -   `订单总金额`: 自动计算的单人订单总成本。
    -   `操作`:
        -   `详细` 按钮: 点击后打开 "详细信息弹窗"。

### 2.4 详细信息弹窗 (Details Modal)

-   **触发**: 点击人员数据表中对应行的 "详细" 按钮。
-   **内容**:
    -   以更清晰的结构展示此人付款的详细分类。
    -   "劳务报酬部分", "经营所得部分", "个体户部分" 的金额输入框是**可编辑**的。
    -   其他费用和税金字段根据可编辑金额实时计算并显示。
-   **操作**:
    -   `保存` 按钮: 将弹窗内的修改应用到主数据表中，并关闭弹窗。
    -   `取消` 按钮: 放弃所有修改，关闭弹窗。

### 2.5 页面操作栏 (Page Actions)

-   **左侧 (次要操作)**:
    -   `批量操作`: (功能待定)。
    -   `下载支付明细`: 下载包含当前所有人员明细的报表。
    -   `作废订单`: 取消整个订单，此为危险操作，可能需要二次确认。
-   **右侧 (主要操作)**:
    -   `上一步`: 返回上一个流程页面。
    -   `下一步`: 保存当前状态并进入 "发放明细" 页面。
